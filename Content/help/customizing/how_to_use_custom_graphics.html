How to use the custom graphics

In some times need to replace the global images by your own. You can do it without overwriting of the images. You can simulate many of the items you will need to make your game episode unique and colorful on your own.

The process is quite simple, but for those of you who don't know how, here is an in depth look at using custom graphics.

 

Level and episode folders

For customize graphics you don't need replace global data, you can put your graphics in the per level or per episode folder of your game. All level files of the episode placed together in the one folder.

Level Custom Folders - is a folders which have same name which have level file. For example, we have level "Dracula.lvlx", and level custom folder will be a "Dracula".

 

 

If you put custom graphic files with level files together - will be customized to all other levels in this episode, but when you will put your graphics in the level custom folder, the graphics will be customized for this level only.

Also you can replace graphics in the world map. You can create the World Custom Folder, the folder which have same name which this world map file is have. Note: if you will play your episode by SMBX game engine, you shouldn't make world custom folder, because SMBX is not support custom folders for world map. You can place your graphics in the episode folder with world map file together.

 

Customizing of the level graphics

Blocks. To replace the blocks you need create graphics files with names like "block-*.ext". Where * - is a ID of block which you is relpacing. Where the "ext" is an image file extension. The SMBX using a GIF format without transparency. If you want use the transparency, you should to create the transparency mask for your image. The mask image will have name like "block-*m.ext". Example: "block-23.gif" for source image and "block-23m.gif" for mask image to define transparency pixels. (Read more about Mask images). The filename of target image file for customizing you can look in the global image directory of your config pack. For the SMBX you should use the GIF format with mask images.

The blocks have a three types of graphics:

- Static

- Animated

- Sizable.

 

   -Static blocks have the one static frame. You can use images with any sizes, but you should use the same format which using your game config which is used for your level and episode.

 

Example of the static image

 

   -Animated blocks have the vertical frameset image. Each frame must have same height and your image must have same frames number which have original block

   -Sizable blocks have one image which deviding into 9 pieces which calling as "Corners", "Sides" and "Center". Sizable blocks can have any size and can be resized. The sizable blocks always displaying under most of BGO's.

    Note If you customizing block for SMBX, image should have image size in the 96x96 pixels.

Background objects. Is a simple scenery of levels. Sceneries have only two types: Static and Animated. BGO images is similar to Blocks, but have one difference: If you will use your episodes for SMBX, images must have the SAME size which have original sprite. The image filesnames of BGO's usually is a "background-*.ext".

 

Static picture

 

Sprite for 4 frames and animated picture

 

 

Be careful for using BGO's: the BGO's shared by Z-position: background-2 Z-level, is a most lower Z-level, these BGO's displaying under sizable blocks. Background-1 Z-level is a displaying over sizable blocks, but under playable character and NPC's and blocks. The foregound level is a displaying over anything, but under blocks and BGO's which the "foreground-2" Z-level. Note for SMBX: In the SMBX BGO's defining its Z-positiob by array data position. I.e. each BGO have own sorting priority, which defining the BGO's Z-position.

 

 

Non-Playable Characters have the own graphics standard. You should use custom graphics with same size and with same animation frames on the sprite. If you want to use custom frames number, custom size, you can define all graphical settings in the special config file, which naming as NPC.txt (read more). The image filenames of NPC's usually is a "npc-*.ext".

NPC have multiple graphics types:

- Statical, you can use one image with one frame for item.

 

- Animated simple, have animated frames like blocks or BGO.

 

 

- Animated with left-right direction defining. Allows to define images for left and right NPC's direction.

    

 

- Animated with left-right-upper directions defining, Allows to define images for left and right NPC's directions and also left

 and right directions for upper state.

    

 

- Animated with rotation. Allows to define NPC's frames with round rotation (NPC should can adhease to celling and walls).

     

 

- Animated with special animation algorithm. The NPC have algorithmic pre-defined frame sequance, and you can't redefine them if you will use your NPC in the original SMBX.

  

 

(Read more about NPC customization)

 

 

Playable Characters. Is a 10x10 matrix-sprite which have size 1000x1000 pixels and 100 frames totally. Each frame of playable character can have maximum of size is 100x100 pixels. When you customizing playable character, the frames must be calibrated to SAME physics config which have the original frameset of your playable character. Note: SMBX haven't possibles for edit collision (hit) boxes for playable character's frames, and you should use predefined collision boxes positions for draw your sprite of your character.

 

 

 

 

Backgrounds - Is the images which displaying under all your items on level. The backgrounds usually have image filenames like "background-2-*.ext".

Backgrounds have two types if images: statical and animated like BGO's, but backgrounds have own displaying modes: single-row, double-row and tiled.

 

- single-row backgrounds have one row which repeating horisontally, but is not repeating vertically.
The single-row backgrounds moving vertically proportionally to section's height.

Image height must be more than 600 pixels (default section height).

Screen only and real demo (vertical)

 

 

- double-row backgrounds is a composite background which use two images for each row.
The double-row background is not moving vertically.
Summary height of both images must be more than 600 pixels (default section height).

 

Screen only and real demo

  

Note for the editor: Single-row and Double-row backgrounds will be displayed only in the bottom or in the top (which attachment type have this background) with only horisontal repeating. If you want to see your background like tiled, you need when you exporting image set the "Tiled background" flag. Also in the INI configs available flag which called as "tiled in the editor" which will make your single-row background like tiled, but in editor only, in game process your background will be displayed in the normal mode.

 

Be careful, if you replacing the single-row backgounds: they are repeats by horisontal only, and in the editor you will see the attached image. You can use background images with any sizes, but when you customizing the animated background, it's sprite must have the same frames number which have original background. Also, the background image must have height is at least height of screen (in the SMBX is screen height is a 600 pixels).

 

 

 

- tiled background repeats by both directions: horisontally and vertically. Image can have any sizes.

 

 

Note: Some backgrounds have the magic effect, which deviding your image into strips which moving with diffirence speeds.

 

 

 

 

Customizing of the world graphics

World graphics usually placing together with your world map file, but you can put all images in the World Custom Directory (not supported by SMBX).

Tiles, Paths, Sceneries. The graphics similar to the BGO for levels. Images have a two types: statically and animated. Image files usually have names "tile-*.ext", "path-*.ext" and "scene-*.ext".

Levels. Level images have tiny diffirence from other images: image always attaching to bottom-center position of item, where image was used. The image files of levels usually is a "level-*.ext".

 

 

 

Customizing notes

- On customizing of animated items be careful, your sprite must be have the same frames number which in the original image.

- If you creating the custom BGO's, please, be careful by using these graphics in the SMBX. If size of custom BGO larger than original item image which you replacing, it will be cropped in the SMBX. Don't use larger images if you wanna use levels in the SMBX. Same for a Tiles, Sceneries, Paths and Levels on the world map.

- On customizing NPC's, you must create the image sprite with same image size and frames number. If you create custom image size, you should to create the npc-*.txt config file for define the properties for your custom NPC.

- Be careful with filenames on the UNIX-Like Operating systems, files with names "thecat" and "TheCat" - is a difference files.

 

 

 

Graphics formats support

 

The PGE support these graphics formats:

Format Description Notes
GIF CompuServe GIF, witout transparency (with using of the masks)
GIF CompuServe GIF, with own transparency SMBX will fill transparency by black
BMP Uncompressed image format (with using of the masks)
PNG Portable Network Graphics, with alpha-channel transparency Not support by SMBX
JPG  Joint Photographic EG, Lossy-data compressed Not support by SMBX
TIF Tagged Image File Format Not support by SMBX
TGA Truevision TGA Not support by SMBX
